## わかったこと
- 管理画面の Basic 認証は学習用途の固定ID/PW実装のまま公開するのは危険であり、環境変数管理に切り替えるのが妥当
- README にはなぜその設計にしたかまで書くと後から見たときの判断理由が残る
- view に残すべき処理と、services / utils に切り出すべき処理の判断基準が明確になった  
  - HTTPの出入口（request / response）を扱うかどうかが境目
- request を受け取っていても、HTTPレスポンスを組み立てていなければ services に置いて問題ない
- 「HTTPの出入口以外は view に置かない」というルールで考えると迷いが減る

## 感じたこと、メモ
- ECサイトは大きな機能追加は終わったが、公開を意識すると細かい部分が気になり微調整などをしてやっといい感じになった
- 設計の判断をブログや README に言語化したことで、なんとなくやった実装が減り自分でも振り返りやすくなった
