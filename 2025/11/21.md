## わかったこと
- Django のモデルはクラスで、レコードはオブジェクト。`ForeignKey` は他オブジェクトへの参照で `order` と `order_id` の違いがオブジェクト指向そのもの。
- `related_name` により `order.items.all()` のような逆参照が可能でデータ操作が OOP 的に自然に行える。
- `__str__` メソッドはオブジェクト自身の文字列表現を定義する仕組みで、OOP の基本を Django でも実感した。
- Django の ORM は「RDB を OOP として扱うフレームワーク」であると整理できた。
- `ImageField` 利用時には `Pillow` が必須で未インストールだと起きるエラーの原因と解決手順を理解した。
- Django × Docker では makemigrations / migrate をDocker コンテナ内で実行する必要があった（うっかりローカルで実行してエラー発生）
