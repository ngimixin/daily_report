## わかったこと
- Djangoの  
 `static` は「開発者が最初から用意しておくファイル」。  
`media` は「サイト運営者が後から追加するデータ」。
- `MEDIA_ROOT` は保存場所、`MEDIA_URL` はアクセス用 URL という役割の違いがある。
- `MEDIA_ROOT / MEDIA_URL` の両方を設定して初めて画像アップロードと表示が成立する。
- Postgres 18 の仕様変更と旧データの不整合によりコンテナが起動しなかった。16 に固定することでテンプレと整合した。
- Docker のDBコンテナはバージョン固定が必須で、`latest` はトラブルの元になりやすいらしい。
- Git がpushできなかった原因は `origin` がHCテンプレを向いたままだったため。自分のSSHリポジトリへ付け替える必要があった。
