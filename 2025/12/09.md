## わかったこと
- context processor はサイト名やカート個数など、全ページで必要な共通データを自動でテンプレートに渡す仕組みであり、ビューで毎回渡す必要がなくなる
- ナビバーのカート個数やサイト名など、全ページで必要なデータは context processor にまとめることで設計が整う
- config に置く理由はアプリ横断の共通処理
- aggregate() の戻り値は辞書
- `dict.get()` を使う理由はキーが存在しない場合にエラーではなく None を返すため
- 合計が None のとき `or 0` とすることで、空カートでも扱いやすい値に変換できる
- aggregate は「全体の集計」、annotate は「行ごとの集計値付与」と役割が別。
