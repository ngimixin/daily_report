## わかったこと
- `(value || "").replace(/\D/g, "")` は「数字以外を全部削除する」ための超定番ワンライナーで、`null / undefined` 対策も同時にできる
- JavaScript の `replace(/\D/g, "")` と Python の `"".join(ch for ch in value if ch.isdigit())` は発想が同じ
- オプショナルチェイニング `?.` は「要素が存在しない場合のクラッシュ防止」で、値そのものを保証するものではない
- `?.` と `|| ""` は役割が違い、`trim()` を安全に呼ぶには両方必要
- `trim()`（JS）と `strip()`（Python）はどちらも「前後の空白だけ」を削除するメソッド
- 郵便番号のように数字だけの場合、`encodeURIComponent` は必須ではないが保険として付けることが多い
